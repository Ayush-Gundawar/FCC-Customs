Sass Introduction

What is Sass?

  • Sass stands for Syntactically Awesome Stylesheet
  • Sass is an extension to CSS
  • Sass is a CSS pre-processor
  • Sass is completely compatible with all versions of CSS
  • Sass reduces repetition of CSS and therefore saves time
  • Sass was designed by Hampton Catlin and developed by Natalie Weizenbaum in 2006
  • Sass is free to download and use
Sass File Type Sass files has the ".scss" file extension. Sass Comments Sass supports standard CSS comments /* comment */, and in addition it supports inline comments // comment:

Sass Variables

With Sass, you can store information in variables, like:

  • strings
  • numbers
  • colors
  • booleans
  • lists
  • null
Sass uses the $ symbol, followed by a name, to declare variables: $variablename: value; Sass Variable Scope Sass variables are only available at the level of nesting where they are defined. Using Sass !global !global indicates that a variable is global, which means that it is accessible on all levels. Tip: Global variables should be defined outside any rules. It could be wise to define all global variables in its own file, named "_globals.scss", and include the file with the @include keyword.

Sass Nested Rules and Properties

Sass Nested Rules Sass lets you nest CSS selectors in the same way as HTML. ex: nav {
--- ul {
-------- margin: 0;
-------- padding: 0;
-------- list-style: none;
--- }
--- li {
-------- display:
inline-block;
--- }
--- a {
-------- display: block;
-------- padding: 6px 12px;
-------- text-decoration: none;
--- }
}
 Notice that in Sass, the ul, li, and a selectors are nested inside the nav selector. Because you can nest properties in Sass, it is cleaner and easier to read than standard CSS. Sass Nested Properties Many CSS properties have the same prefix, like font-family, font-size and font-weight or text-align, text-transform and text-overflow. With Sass you can write them as nested properties

Sass @import and Partials

Sass keeps the CSS code DRY (Don't Repeat Yourself). You can create small files with CSS snippets to include in other Sass files. Sass Importing Files The @import directive allows you to include the content of one file in another. no extra HTTP call is required at runtime. @import filename; Sass Partials If you start the filename with an underscore, Sass will not transpile it. Files named this way are called partials in Sass. Sass Partial Syntax _filename; Now, if you import the partial file, omit the underscore.

Sass @mixin and @include

Sass Mixins The @mixin directive lets you create CSS code that is to be reused throughout the website. The @include directive is created to let you use (include) the mixin. Defining a Mixin Sass @mixin Syntax: @mixin name {
--- property: value;
--- property: value;
--- ...
}
In sass Hyphens and underscores are considered to be the same. Using a Mixin Sass @include mixin Syntax: selector {
--- @include mixin-name;
}
A mixin can also include other mixins Passing Variables to a Mixin mixin with arguments. @mixin example($param1, $param2) {
--- property: $param1 $param2 ...;
}
.myContainer {
--- @include example(blue, 1px);
}
Default Values for a Mixin @mixin example($param1: value1, $param2: value2) {
--- property: $param1 $param2 ...;
}
you need to specify the values that change when you include the mixin. Another good use of a mixin is for vendor prefixes.

Sass @extend and Inheritance

Sass @extend Directive The @extend directive lets you share a set of CSS properties from one selector to another. The @extend directive is useful if you have almost identically styled elements that only differ in some small details.The @extend directive helps keep your Sass code very DRY. .myContainer{
--- property1: value1;
--- property2: value2;
--- ...
}
.myContainer2{
--- @extend .myContainer1;
--- property3: value3;
--- ...
}
This code is equivalent to css's: .myContainer, .myContainer2{
--- property1: value1;
--- property2: value2;
--- ...
}
.myContainer2{
--- property3: value3;
}

Sass String Functions

Sass String Functions Sass strings are 1-based. The first character in a string is at index 1, not 0.

  • quote(string)
  • str-index(string, substring)
  • str-insert(string,insert_string, index)
  • str-length(string)
  • str-slice(string, start, end)
  • to-lower-case(string)
  • to-upper-case(string)
  • unquote(string)

Sass Numeric Functions

  • abs(number)
  • ceil(number)
  • comparable(num1, num2) : compares unit of numbers ex: px, cms, etc..
  • floor(number)
  • max(num1, num2, ...)
  • min(num1, num2, ...)
  • random()
  • round()

Sass List Functions

Sass lists are immutable (they cannot change). So, the list functions that return a list, will return a new list, and not change the original list. Sass lists are 1-based. The first list item in a list is at index 1, not 0.

  • append(list, value, seperator): seperator is optional
  • index(list, value)
  • join(list1, list2, [seperated])
  • length(list)
  • list-seperator(list)
  • nth(list, n)
  • set-nth(list, n)
  • zip(list1, list2, ...): Combines lists into a single multidimensional list.

Sass Map Functions

Sass Map Functions It is also possible to use the List functions from the previous page, with maps. Then the map will be treated as a list with two elements. Sass maps are immutable. So, the map functions that return a map, will return a new map, and not change the original map.

  • map-get(map, key)
  • map-has-key(map, key)
  • map-keys(map)
  • map-merge(map1, map2)
  • map-remove(map, keys...)
  • map-values(map)

Sass Introspection Functions

Sass Introspection Functions The introspection functions are rarely used when building a stylesheet. However, they are valuable if something does not work properly - to figure out what's going on: like debugging functions. Examples:

  • call(function, arguments...)
  • content-exists()
  • feature-exists(feature)
  • function-exists(functionname)
  • get-function(functionname, css: false),
  • global-variable-exists(variablename)
  • inspect(value)
  • mixin-exists(mixinname)
  • type-of(value)
  • unit(number)
  • unitless(number)
  • variable-exists(variablename)
